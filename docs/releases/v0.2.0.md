# ReplayBench-IoT v0.2.0

ReplayBench-IoT turns the original IoT replay-defense simulator into a reproducible benchmark platform with corrected anti-replay semantics, confidence intervals, realistic channel/cost models, adaptive defenses, and public-facing packaging.

## Behavior Changes

- Sliding-window mode now follows RFC-style right-edge advancement instead of rejecting far-future authenticated counters.
- Challenge-response now supports multiple outstanding nonces with replay tracking, TTL, and caps.
- API simulation responses now return `SimulationSpecPublic` and never echo `shared_key`.
- Web builds use webpack to avoid Turbopack path issues on non-ASCII workspaces.

## Reproducibility

```bash
PYTHONPATH=src:. python3 -m pytest -q
ruff check .
mypy src
npm --prefix web run lint
npm --prefix web run test:contracts
npm --prefix web run build
```

## Known Limitations

- Trace-driven loss accepts `list[bool]` traces only; pcap/CSV ingestion is not implemented.
- NISTIR 8259A and ETSI EN 303 645 references are narrative alignment only.
- Hardware validation artifacts cover controlled lab links and are not a certification claim.
